# Инструкция по запуску сайта-приглашения

Внутри архива:

- `index.html` + папка `assets/` — сам сайт (открыть/захостить)
- `Code.gs` — код для Google Apps Script (бэкенд)
- `SETUP.md` — этот файл

Сайт уже наполнен вашим контентом и дизайном. Осталось подключить базу данных (Google Таблицу) и захостить сайт.

---

## Шаг 1. Создайте Google Таблицу

1. Зайдите на [sheets.google.com](https://sheets.google.com) → «Новая таблица».
2. Переименуйте таблицу как хотите (например, «Свадьба — гости»).
3. Создайте **две вкладки** с названиями **строго** `groups` и `people` (маленькими латинскими буквами).

### Вкладка `groups`

В первой строке (шапка) впишите заголовки столбцов **ровно в таком порядке**:

| A | B | C | D | E | F | G | H |
|---|---|---|---|---|---|---|---|
| group_id | names | children_allowed | creation_date | last_update_date | will_come_group | hello_form | decline_reason |

Дальше — по одной строке на каждую группу гостей (= одна персональная ссылка). Пример:

| group_id | names | children_allowed | creation_date | last_update_date | will_come_group | hello_form | decline_reason |
|---|---|---|---|---|---|---|---|
| semya_ivanovyh | Анастасия и Александр Ивановы | 1 | | | | Дорогие Анастасия и Александр | |
| druzya_1 | Друзья со студенчества | 0 | | | | Дорогие друзья | |

Пояснения:
- **group_id** — уникальный код группы, он же используется в ссылке `?guest=group_id`. Можно писать латиницей и цифрами, без пробелов.
- **children_allowed** — `1`, если группе можно приходить с детьми, `0` — если нет. Именно от этого поля зависит, появится ли в разделе «Детали» просьба оставить детей дома (она показывается, когда стоит `0`).
- **hello_form** — как обратиться к гостям на приветственном экране, например «Дорогие Анастасия и Александр» (без запятой в конце — сайт сам добавит нужный текст).
- **creation_date, last_update_date, will_come_group, decline_reason** — оставьте пустыми, сайт заполнит их сам после ответа гостей.

### Вкладка `people`

Шапка:

| A | B | C | D | E | F | G | H | I | J | K | L | M | N | O | P | Q | R |
|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|
| group_id | personal_id | name | last_name | first_name | patronymic | will_come_personal | drink | drink_other | no_alcohol_pref | no_alcohol_pref_text | allergy | allergy_text | hot_food | transport_to | stay_second_day | sleep_place | transport_from | transport_from_option |

(да, это 19 столбцов — просто продолжайте по алфавиту в ту же строку)

**Важно:** для каждой группы заранее добавьте по одной строке на каждого ожидаемого гостя, заполнив только `group_id`, `personal_id` и `name` — остальное гость заполнит сам через анкету на сайте.

Пример для группы `semya_ivanovyh`:

| group_id | personal_id | name | last_name | ... |
|---|---|---|---|---|
| semya_ivanovyh | p1 | Анастасия | | |
| semya_ivanovyh | p2 | Александр | | |

- **personal_id** — любой уникальный код в рамках всей таблицы (например `p1`, `p2`, `p3`…).
- **name** — привычное имя, по которому в анкете к гостю обратятся: «Уважаемый(ая), Анастасия, заполните форму опроса ниже».

## Шаг 2. Установите скрипт-бэкенд (Google Apps Script)

1. В таблице откройте **Расширения → Apps Script**.
2. Удалите содержимое файла `Code.gs`, которое там по умолчанию, и вставьте туда содержимое файла `Code.gs` из архива.
3. Сохраните (значок дискеты).
4. Нажмите **Развернуть → Новое развёртывание** (Deploy → New deployment).
5. Тип — **Веб-приложение** (Web app).
6. «Execute as» (Выполнять от имени) — **Me / От своего имени**.
7. «Who has access» (У кого есть доступ) — **Anyone / Все**.
8. Нажмите «Развернуть», разрешите доступ (потребуется подтвердить через ваш Google-аккаунт — появится предупреждение «Google не проверял это приложение», это нормально, нажмите «Дополнительно» → «Перейти на страницу (небезопасно)», это ваш собственный скрипт).
9. Скопируйте выданный **URL веб-приложения** — он выглядит примерно так:
   `https://script.google.com/macros/s/AKfycb.../exec`

Если вы потом ещё будете редактировать код `Code.gs` — при каждом изменении нужно делать **новое** развёртывание (или «Управление развёртываниями → изменить версию»), иначе правки не применятся.

---

## Шаг 3. Впишите URL в сайт

1. Откройте файл `assets/wedding.js` в любом текстовом редакторе.
2. В самом начале файла найдите строку:
   ```js
   var APPS_SCRIPT_URL = 'ВСТАВЬТЕ_СЮДА_URL_ВАШЕГО_APPS_SCRIPT_WEB_APP';
   ```
3. Замените на ваш реальный URL из шага 2, например:
   ```js
   var APPS_SCRIPT_URL = 'https://script.google.com/macros/s/AKfycb.../exec';
   ```
4. Сохраните файл.

Без этого шага сайт будет открываться и выглядеть как надо, но анкета не будет ни подгружать, ни сохранять данные.

## Шаг 4. Разместите сайт в интернете

Персональные ссылки вида `?guest=...` работают только если сайт лежит по реальному адресу (не просто открыт как файл на компьютере). Самый простой бесплатный вариант — **GitHub Pages**:

1. Создайте новый репозиторий на [github.com](https://github.com), например `wedding`.
2. Загрузите туда `index.html` и всю папку `assets/` (через веб-интерфейс: «Add file → Upload files», перетащите всё содержимое, включая папку `assets`).
3. В репозитории: **Settings → Pages → Source** выберите ветку `main` и папку `/root` → Save.
4. Через 1–2 минуты сайт будет доступен по адресу вида:
   `https://ваш-логин.github.io/wedding/`

Подойдёт и любой другой хостинг (Netlify, Vercel, обычный хостинг с загрузкой по FTP) — просто загрузите `index.html` и папку `assets` в корень.

## Шаг 5. Персональные ссылки для гостей

Ссылка для конкретной группы формируется так:

```
https://ваш-домен/?guest=GROUP_ID
```

Например, для группы `semya_ivanovyh`:

```
https://ваш-логин.github.io/wedding/?guest=semya_ivanovyh
```

Если гость откроет сайт **без** параметра `?guest=...` (или с несуществующим значением) — увидит стандартный текст «Дорогие друзья…» и обычную чистую анкету без предзаполненных имён.

## Что важно знать об уже готовом сайте

- **Все ваши изображения** (фото жениха/невесты, календарь, фон для блока с таймером, голубки на заставке) подключены напрямую по вашим ссылкам на GitHub (`raw.githubusercontent.com/yudyymal/wedding_invitation/...`). Если вы удалите или переименуете файлы в том репозитории — они пропадут и на сайте. Если хотите, эти картинки можно вместо этого положить прямо в папку `assets` — тогда сайт не будет зависеть от внешнего репозитория.
- **Карта.** Точного бесплатного встраивания интерактивной карты 2ГИС без API-ключа не существует — 2ГИС требует ключ для своего JS-виджета (MapGL). Поэтому по кнопке «Карта» сайт показывает встроенную карту OpenStreetMap с меткой на нужных координатах, а рядом — кнопка «Открыть в 2ГИС», которая откроет точку по координатам 56.807015, 60.793849 в приложении/на сайте 2ГИС (на телефоне откроется приложение 2ГИС, если оно установлено). Если вы захотите именно встроенный виджет 2ГИС — заведите бесплатный API-ключ на [dev.2gis.com](https://dev.2gis.com), и я (или любой разработчик) быстро подключу его вместо OpenStreetMap.
- **Блок «оставьте детей дома».** В вашем ТЗ было два взаимоисключающих описания условия показа этого блока. Я реализовал по смыслу самого текста: блок появляется, когда `children_allowed = 0` (детям нельзя), и скрыт, когда `children_allowed = 1` (детям можно). Если нужно наоборот — в файле `assets/wedding.js` в функции `applyChildrenBlock` достаточно поменять `=== 0` на `=== 1`.
- **Контакты.** Жених — Telegram (t.me/AlexanderNorthy), невеста — MAX (max.ru). Если нужно оба мессенджера у каждого — легко добавить, дайте знать соответствующие ссылки.
- **Музыка** и блок со значком её включения/выключения полностью убраны, как и просили.
- Заголовки разделов теперь набраны шрифтом **Great Vibes** (Google Fonts) вместо картинок — при открытии сайта без интернета шрифт не подключится и подставится обычный курсив браузера, но с интернетом всё будет как задумано.

## Проверка анкеты без интернета/таблицы

Пока `APPS_SCRIPT_URL` не заполнен, при попытке отправить анкету появится предупреждение об этом — это ожидаемое поведение, а не баг. После настройки шагов 1–3 всё заработает.
